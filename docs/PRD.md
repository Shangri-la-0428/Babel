# BABEL — 产品需求文档 (PRD)

> AI-Driven World State Machine
> 版本: 1.0 | 最后更新: 2026-03-23

---

## 一、产品愿景

BABEL 是一个 **AI 驱动的世界状态机**。用户播下一颗"世界种子"（Seed），AI 运行时引擎接管一切——角色自主行动、对话、交易、移动，涌现出不可预测的叙事。

**核心隐喻**：用户是观察者，透过终端屏幕凝视一个活着的微型世界。不是写故事，而是**观察故事自发生长**。

### 目标用户

创意型世界构建者——作家、游戏设计师、AI 爱好者。他们希望：
- 以最少配置启动一个鲜活的模拟世界
- 观察 AI 角色之间的自发互动与涌现剧情
- 在关键时刻干预（注入事件、与角色对话）
- 提取有价值的叙事片段作为可复用资产

### 核心价值主张

| 维度 | 描述 |
|------|------|
| **即时启动** | 选择预设种子 → 一键创建 → 世界开始运行 |
| **涌现叙事** | 每个角色由 LLM 独立驱动决策，无预设剧本 |
| **实时观察** | WebSocket 推送事件流，粒子系统 + 雷达可视化 |
| **可干预** | 注入世界事件、与角色对话、添加新角色 |
| **可积累** | 提取并保存角色/物品/地点/事件为可复用种子 |

## 二、系统架构

```
┌─────────────────────────────────────────────┐
│                  Frontend                    │
│  Next.js 14 (App Router) + Tailwind CSS     │
│  4 Routes: / | /sim | /create | /assets     │
└───────────────┬────────────┬────────────────┘
                │ REST API   │ WebSocket
┌───────────────┴────────────┴────────────────┐
│                  Backend                     │
│  FastAPI + SQLite + litellm                  │
│  Engine Loop → LLM → Event → Broadcast      │
└─────────────────────────────────────────────┘
```

**技术栈**：
- **前端**: Next.js 14, Tailwind CSS 3.4, TypeScript, Canvas 2D (粒子系统 + 雷达)
- **后端**: Python FastAPI, SQLite, litellm (多 LLM 提供商)
- **实时通信**: WebSocket (`/ws/{session_id}`)
- **设计系统**: 赛博朋克暗色主题，JetBrains Mono，零圆角，石灰绿 (#C0FE04) 单色强调

### 模块 M5: AI 充实系统 (Enrichment)

#### M5.1 实体充实

| ID | 功能 | 描述 |
|----|------|------|
| M5.1.1 | 角色充实 | LLM 基于事件历史生成：背景故事、性格特征、关系网络 |

| M5.1.4 | 增量更新 | 已有详情作为上下文传入 LLM，生成增量补充（不覆盖） |
| M5.1.5 | 持久化存储 | 生成结果存入 `entity_details` 表，不会丢失 |

### 模块 M7: 引擎核心

> 以下为后端逻辑，但前端可通过 API 和 WebSocket 感知其行为。

#### M7.1 Tick 推演循环

| ID | 功能 | 描述 |
|----|------|------|
| M7.1.1 | Tick 递增 | 每轮 `session.tick += 1` |
| M7.1.2 | 角色顺序 | 随机打乱存活角色顺序 |
| M7.1.3 | 配角跳过 | supporting 角色约 30% 概率跳过本轮 |
| M7.1.4 | LLM 动作决策 | 构建 prompt → 调用 LLM → 解析 JSON 响应 → 验证动作合法性 |
| M7.1.5 | 6 种动作类型 | speak（对话）, move（移动）, use_item（使用物品）, trade（交易）, observe（观察）, wait（等待） |
| M7.1.6 | 状态变更 | 动作可能改变：角色位置、物品栏（增减物品） |
| M7.1.7 | 事件生成 | 每个角色动作生成一条 Event，包含：tick、角色、动作类型、结果描述 |
| M7.1.8 | 事件广播 | 通过 WebSocket 实时推送每条事件 |
| M7.1.9 | 中途暂停 | 运行中可随时暂停，当前 tick 内剩余角色不再执行 |

#### M7.2 反重复系统

| ID | 功能 | 描述 |
|----|------|------|
| M7.2.1 | 重复检测 | 检测角色连续 3 次执行相同动作类型且内容相似 |
| M7.2.2 | LLM 扰动 | 生成意外世界事件打破循环（如"一个陌生人到来"） |
| M7.2.3 | 模板回退 | LLM 生成失败时，使用 12 个预设模板 |

#### M7.3 记忆系统

| ID | 功能 | 描述 |
|----|------|------|
| M7.3.1 | 滑动窗口 (Legacy) | 每个角色保留最近 10 条记忆摘要 |
| M7.3.2 | 结构化记忆 | 4 类：episodic（经历）, semantic（知识）, social（社交）, goal（目标） |
| M7.3.3 | 重要性评分 | 基于动作类型 + 目标关联度 (0.1–0.9) |
| M7.3.4 | 上下文检索 | 加权公式：0.3×时效性 + 0.35×重要性 + 0.25×相关性 + 0.1×访问衰减 |
| M7.3.5 | 记忆整合 | 每 5 轮：将关于同一角色的 3+ 条 episodic 记忆合并为 1 条 semantic 记忆 |

#### M7.4 时间线与快照

| ID | 功能 | 描述 |
|----|------|------|
| M7.4.1 | TimelineNode | 每 tick 创建一个节点（摘要、事件数、角色位置、是否重要） |

## 八、已知限制与计划功能

### 当前限制
- 仅支持桌面端 (1280px+)
- 无用户认证系统
- 每个种子最多 3-4 个角色（LLM 成本限制）
- 时间线分叉（Branch）功能已部分搭建但未完成

### 计划中功能 (参考 Plan)
- **Phase 1**: 世界时钟系统 — 虚拟世界时间 ✅ (已实现)
- **Phase 2**: 时间线分叉 — 从过去某 tick 分叉平行时间线 (设计完成，待开发)
- **Phase 3**: 定时事件 — 种子中配置自动触发的世界事件 (待设计)
