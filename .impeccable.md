## Design Context

### Users
Creative worldbuilders — writers, game designers, and AI enthusiasts who use BABEL to seed emergent narratives and observe AI agents interact in simulated worlds. They approach it like peering into a living terrarium: fascination, wonder, the thrill of unexpected emergent behavior. Their context is desktop (1280px+), likely in a focused creative session, wanting to feel like they're operating a portal into another reality.

### Brand Personality
**Dark / Electric / Raw** — Cyberpunk energy. Gritty, alive, on the edge. Industrial with a pulse. Think abandoned space station control room that just powered on: every indicator hums, every data stream is alive. Not polished consumer software — this is a raw instrument for world observation.

### Aesthetic Direction
- **Primary reference**: [marathonthegame.com](https://marathonthegame.com) — sci-fi industrial minimalism, void-black backgrounds, neon lime accent, brutal grid layouts, uppercase monospace typography, 1px hairline separators
- **Anti-references**: Rounded SaaS dashboards, pastel palettes, Material Design, glassmorphism, bounce/elastic easing, anything "friendly" or "approachable"
- **Theme**: Dark mode only. Pure black void (#000000) with lime (#C0FE04) as the singular accent pulse
- **Typography**: Monospace-first (JetBrains Mono), all uppercase by default (`text-transform: uppercase; letter-spacing: 0.05em` on body), Inter for display/headings only
- **Geometry**: Zero border-radius everywhere (`* { border-radius: 0 }`). Status dots are the ONLY circular element in the entire system
- **Grid pattern**: Marathon-inspired 1px gap data grids — cells separated by `gap-px bg-b-DEFAULT` hairlines, not padding
- **Texture**: CRT scanline overlay (`.scanlines::after`) on simulation page for atmospheric depth

### Design Principles
1. **Void-first** — Black is not a background, it's the default state. Content emerges from darkness. Every surface addition must justify its existence against the void.
2. **Signal over decoration** — Every visual element carries information. No ornamental borders, shadows, or gradients. Color means state change. Animation means something happened.
3. **Machine aesthetic** — The interface should feel like it was designed by the system observing itself. Clinical precision, tabular data, monospace readouts. Human warmth is the wrong tone.
4. **One accent, infinite meaning** — Lime (#C0FE04) is the only warm color. It means: active, alive, primary, selected, running. Guard it — overuse kills its signal power.
5. **Density over whitespace** — Pack information tight. Use 1px gaps, compact grids, and small type. The user wants to see everything at once, like a mission control dashboard.

### Technical Constraints
- **Platform**: Desktop only, 1280px+ viewport
- **Stack**: Next.js 14 (App Router) + Tailwind CSS 3.4 with custom preset (`design/tailwind.preset.js`)
- **i18n**: 210+ translation keys in `lib/i18n.ts`, CN/EN toggle. All user-facing strings use `t()` — no hardcoded text
- **Accessibility**: MVP level — semantic HTML, keyboard navigation, `prefers-reduced-motion` respected globally. Focus rings: 2px solid primary with `outline-offset: 2px`. Input focus: border-primary + `box-shadow: 0 0 8px var(--color-primary-glow)`
- **Color tokens**: Semantic naming via Tailwind (`text-t-muted`, `border-b-DEFAULT`, `bg-surface-1`) — NEVER raw Tailwind colors

## Established Component Patterns

### Buttons

**Primary action** (filled):
```
bg-primary text-void border border-primary
hover:bg-transparent hover:text-primary hover:shadow-[0_0_16px_var(--color-primary-glow-strong)]
active:scale-[0.97]
disabled:opacity-30
transition-[colors,box-shadow,transform]
```

**Secondary action** (bordered):
```
border border-b-DEFAULT text-t-muted
hover:border-primary hover:text-primary
active:scale-[0.97]
disabled:opacity-30
transition-[colors,transform]
```

**Tertiary action** (text-only):
```
text-t-muted hover:text-t-DEFAULT transition-colors
```

**Danger action**:
```
bg-danger text-void border border-danger
hover:bg-transparent hover:text-danger
disabled:opacity-30
transition-colors
```

All action buttons: `h-9 px-4-5 text-micro font-medium tracking-wider`

### Form Inputs
```
h-9 px-3 bg-void border border-b-DEFAULT text-detail text-t-DEFAULT
normal-case tracking-normal
focus:border-primary focus:outline-none
hover:border-b-hover
transition-colors
```
Focus glow added globally via `globals.css` — `box-shadow: 0 0 8px var(--color-primary-glow)`

### Form Labels
```
text-micro text-t-muted tracking-widest mb-1.5 block
```
Data display labels (read-only) use `mb-1`.

### Empty States
```html
<div class="border border-b-DEFAULT p-8 flex flex-col items-center gap-4">
  <div class="text-micro text-t-dim tracking-widest">{"// EMPTY"}</div>
  <div class="text-detail text-t-muted normal-case tracking-normal">{description}</div>
  <a class="...primary-or-secondary-button...">{CTA}</a>
</div>
```
The `// COMMENT` label is a signature BABEL pattern — machine-style code comments as empty state indicators.

### Loading Skeletons
```
bg-gradient-to-r from-surface-2 via-surface-3 to-surface-2
bg-[length:200%_100%] animate-[shimmer_1.5s_ease_infinite]
```

### Badges / Type Tags
```
text-micro tracking-wider px-2.5 py-0.5 border leading-none font-medium
```
Color via semantic class: `text-info border-info`, `text-danger border-danger`, etc.

### Navigation
- Breadcrumb pattern: `← 返回 | BABEL / 世界名称` with lime glow on brand and world name
- BABEL brand: always `text-primary`, links get `hover:drop-shadow-[0_0_8px_var(--color-primary-glow-strong)]`
- Active nav item: `text-primary` with `aria-current="page"`
- Inactive nav item: `text-t-muted hover:text-t-DEFAULT`

### Expand/Collapse Sections
Use `animate-slide-down` (translateY(-8px) → 0 + fade, 150ms ease).

### Data Grids (Marathon-style)
```
<div class="flex flex-col gap-px bg-b-DEFAULT">
  <div class="bg-void px-4 py-3">...</div>
  <div class="bg-void px-4 py-3">...</div>
</div>
```
1px hairline separators via `gap-px` on parent with `bg-b-DEFAULT`.

## Animation System

### Entrance Animations
| Animation | Duration | Usage |
|-----------|----------|-------|
| `fade-in` | 300ms ease | Page content, general reveals |
| `slide-up` | 300ms ease-out | Card/element entrance |
| `slide-down` | 150ms ease | Expand sections, error banners, settings panel |
| `stagger-in` | 200ms + 40ms delays | List items cascading entrance (up to 9 children) |

### Feedback Animations
| Animation | Duration | Usage |
|-----------|----------|-------|
| `active:scale-[0.97]` | instant | Button press tactile feedback |
| `event-flash` | 1.2s ease | New event lime highlight |
| `event-flash-danger` | 1.2s ease | World event red highlight |
| `tick-bump` | 300ms ease-out | Tick counter scale pulse |

### Ambient Animations
| Animation | Duration | Usage |
|-----------|----------|-------|
| `pulse-glow` | 2s ease infinite | Running status dot (24px spread, primary-glow-strong) |
| `shimmer` | 1.5s ease infinite | Loading skeleton gradient sweep |
| `blink` | 1s step-end infinite | "Thinking..." indicator |

### Glow Effects
- Subtle: `shadow-[0_0_8px_var(--color-primary-glow)]` — input focus, small buttons
- Medium: `shadow-[0_0_12px_var(--color-primary-glow)]` — inject button hover
- Strong: `shadow-[0_0_16px_var(--color-primary-glow-strong)]` — primary button hover, Run button
- Hero: `drop-shadow-[0_0_24px_var(--color-primary-glow-strong)]` — hero title
- Inset: `shadow-[inset_0_0_24px_var(--color-primary-glow)]` — world list item hover

### Tab Content Switching
`key={activeTab}` on tab panel container forces React remount → `animate-[fade-in_100ms_ease]`

### Oracle Animations
| Animation | Duration | Usage |
|-----------|----------|-------|
| `oracle-slide-left` | 250ms out-expo | Oracle message entrance (left slide + fade) |
| `oracle-slide-right` | 200ms out-expo | User message entrance (right slide + fade) |
| `oracle-chromatic-in` | 350ms out-expo | Oracle reply entrance (chromatic aberration + slide) |
| `oracle-border-pulse` | 2s ease infinite | Drawer border pulse during thinking (α 0.15–0.6 swing) |
| `transmission-sweep` | 400ms out-expo | Input send flash (scaleX 0→1 + fade) |

### Oracle Visual Presence
- **Edge scan**: 3px beam, 0.7α idle, full 1.0α + `drop-shadow` glow during thinking
- **Particles**: α 0.03–0.07 idle, up to 0.14 cap; thinking ×2.8 speed, ×2.5 brightness
- **Messages**: `bg-info/[0.05]` surface, `border-l-info/30` accent — stronger than standard chat
- **Empty state hero**: `text-sm` label with gradient signal lines + 16px info glow halo
- **Header**: `border-info/15` identity border, label glow, tick in `text-info/40`

### Rules
- All animations respect `prefers-reduced-motion: reduce` via global CSS rule
- Never use bounce or elastic easing — they feel dated
- Exit animations should be ~75% of entrance duration
- Easing: `ease-out` (cubic-bezier(0, 0, 0.2, 1)) for entrances, `ease` for ambient
- `ease-out-expo` (cubic-bezier(0.16, 1, 0.3, 1)) for drawer slide and Oracle message entrances
